# 群聊数据 + LIMS 数据统计分析与可视化平台 — 项目总结

## 项目背景

基于现有的「企业微信客户群聊智能分析系统」扩展，接入 LIMS 系统售后数据，与原始群聊数据关联，实现综合的统计分析与 Web 可视化报告。已在现有 Python FastAPI 服务中完成全部 4 个阶段的开发。

---

## 核心数据流

```
qxChat API ── GET /qxChat/
    │
    ├── 按 roomid 分组 → 提取群名
    │       │
    │       └── 正则提取项目号 LC-P\d+
    │               │
    │               └── POST /unionLims/base_data/ → [{"projectCode":"LC-P2023..."}]
    │                       │
    │                       └── 每条 LIMS 记录:
    │                           ├── finalAfterSaler = afterSaler ∈ members ? afterSaler : ""
    │                           ├── salesPerson = finalAfterSaler ?: afterSaler
    │                           └── is_key_account = keyAccount ≠ ""
    │
    └── 按 projectCode 关联群组与 LIMS 记录 (1:N)
            │
            └── aggregate_report() → 18 个模块 → HTML 报告
```

## 时间维度支持

| 类型 | 时间范围 | 示例 |
|------|----------|------|
| daily | 当日 | 2026-07-03 |
| weekly | 最近 7 天 | 2026-06-27~2026-07-03 |
| monthly | 最近 30 天 | 2026-06-04~2026-07-03 |
| quarterly | 最近 90 天 | 2026-04-05~2026-07-03 |
| yearly | 最近 365 天 | 2025-07-04~2026-07-03 |

---

## 核心派生字段

| 字段 | 计算逻辑 | 来源 | 用于模块 |
|------|----------|------|----------|
| `finalAfterSaler` | afterSaler 在 members 中 → 用它，否则空 | LIMS 单条记录 | M04 / M07 / M11 |
| `salesPerson` | finalAfterSaler 非空 → 用它，否则 fallback | LIMS | M09 |
| `is_key_account` | keyAccount 非空 | LIMS | M07 |

## 维护建议

1. **每日归档**：通过 crontab 每日执行 `curl -X POST /api/v1/report/archive`
2. **磁盘清理**：定期清理老旧归档 JSON 和报告 HTML 文件
3. **数据刷新**：设置定时任务每周全量刷新归档（`?fresh=true`）
4. **ECharts CDN**：如无外网，将 ECharts 库下载到内网静态文件服务器
