# 群聊数据 + LIMS 数据统计分析与可视化平台 — 项目总结

## 项目背景

基于现有的「企业微信客户群聊智能分析系统」扩展，接入 LIMS 系统售后数据，与原始群聊数据关联，实现综合的统计分析与 Web 可视化报告。已在现有 Python FastAPI 服务中完成全部 4 个阶段的开发。

---

## 核心数据流

```
qxChat API ── GET /qxChat/
    │
    ├── 按 roomid 分组 → 提取群名
    │       │
    │       └── 正则提取项目号 LC-P\d+
    │               │
    │               └── POST /unionLims/base_data/ → [{"projectCode":"LC-P2023..."}]
    │                       │
    │                       └── 每条 LIMS 记录:
    │                           ├── afterSaler = LIMS 原始售后人员字段
    │                           ├── salesPerson = saleName / assignmentUser
    │                           └── is_key_account = keyAccount ≠ ""
    │
    └── 按 projectCode 关联群组与 LIMS 记录 (1:N)
            │
            └── aggregate_report() → 18 个模块 → HTML 报告
```

## 数据看板 (Dashboard)

### 功能概述
提供基于 MySQL 每日分析结果和 LIMS 项目数据的交互式 Web 数据看板，访问根路径 `/` 即可进入。

### 数据来源
| 数据 | 来源表 | 关键字段 |
|------|--------|---------|
| 群聊消息趋势 | qx_analysis_result | CREATEDTIME, messageToDayCount, groupName |
| 情感分析 | qx_analysis_result | customerEmotionAnalysis, saleEmotionAnalysis |
| 漏回检测 | qx_analysis_result | isMissedMessage |
| 高频关键词 | qx_analysis_result | highFrequencyWords |
| 销售区域 | t_project / t_income | CREATEDBYORGNAME / orgName |
| 售后人员 | t_income / t_project | afterSaler / AFTERSALER |
| 产品分类 | t_project / t_product_main | PRODUCTBIGSORTONE/TWO/THREE |
| 销售人员 | t_project | CREATEDBYNAME |
| 重点客户 | t_customer / t_income | KEYACCOUNT / keyAccount |

### 看板 API 接口
所有接口位于 `/api/v1/dashboard/` 前缀下。

| 接口 | 方法 | 说明 |
|------|------|------|
| `/overview` | GET | 看板全量数据（KPI + 区域 + 售后 + 产品 + 交叉分析 + 数据质量） |
| `/evidence` | GET | 风险明细数据（漏回/负向情感/高频词），支持分页 |
| `/verify` | GET | **数据验证** — 直接 SQL 查询数据库，返回原始覆盖率与看板值对比 |
| `/export` | GET | **CSV 导出** — 全维度数据导出为 CSV 文件下载 |
| `/cache/clear` | POST | 清除看板缓存 |

### overview 返回结构
```json
{
  "meta": { "period", "start_date", "end_date", "filters", "filter_options" },
  "summary": { "total_groups", "total_messages", "regions", "aftersaler_count", ... },
  "service_quality": { "unanswered": {...}, "sentiment": {...} },
  "communication": { "trend": [...], "high_frequency": [...], "active_duration": [...], "time_period_breakdown": {...} },
  "business": { "aftersalers": [...], "regions": [...], "product_categories": [...], "product_hierarchy": [...], "key_accounts": [...] },
  "cross_analysis": { "region_sales": [...], "region_after": [...], "region_product": [...] },
  "data_quality": { "lims_link_rate", "region_link_rate", "aftersaler_coverage_rate", ... }
}
```

### 数据验证机制
1. **验证 API** (`/api/v1/dashboard/verify`)：直接查询 MySQL 返回 t_project/t_income/qx_analysis_result 三表的原始统计
2. **前端验证面板**：工具栏「数据验证」按钮，展示看板值与数据库直查值对比表
3. **CSV 导出** (`/api/v1/dashboard/export`)：下载完整数据到 Excel，支持所有筛选维度，用于人工核对

### 已知数据约束
- 售后覆盖率取决于 t_income.afterSaler / t_project.AFTERSALER 字段的填充情况
- 区域覆盖率取决于 t_project.CREATEDBYORGNAME / t_income.orgName 字段
- 重点客户取决于 t_customer.KEYACCOUNT 标记（非空且非 0/false/否）
- 时段分布基于固定比例估算（60% 工作时段，50/50 上午/下午拆分），非真实小时粒度

## 开发与部署

### 本地开发
```bash
pip install -e .
uvicorn api.main:app --reload --port 8000
```

### Docker 部署
```bash
docker-compose up -d --build
```

### 环境变量
复制 `.env.production` 为 `.env`，按需修改数据库连接和 LIMS API 地址。

### 依赖
- Python 3.11+
- FastAPI + Uvicorn
- PyMySQL
- httpx (LIMS API 调用)
- ECharts 5 (前端图表)
