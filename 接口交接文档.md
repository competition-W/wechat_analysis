# 企业微信群聊分析服务 - 接口交接文档

## 一、服务概述

企业微信群聊分析服务提供批量群聊消息的智能分析功能，包括情感分析、敏感词检测、摘要生成、高频词统计和漏回消息分析。

**服务地址**: `http://110.1.121.104:8000`

---

## 二、接口列表

### 1. 单群聊分析接口

**Endpoint**: `POST /api/v1/chat/analyze`

**请求示例**:
```json
{
    "room_id": "群聊ID",
    "room_name": "群聊名称（可选）",
    "messages": [
```

## 三、数据字段说明

### 3.1 成员信息 (members)

| 字段 | 类型 | 说明 |
| :--- | :--- | :--- |
| userid | string | 用户唯一标识 |
| name | string | 真实姓名（优先使用） |
| group_nickname | string | 群昵称 |
| type | int | 用户类型：**1=员工，2=客户**（推荐使用此字段判断身份） |
| job | string | 职位：售后、销售等（可选，仅作参考） |
| position | string | 职位描述 |

> **重要说明**：员工/客户身份判断优先使用 `type` 字段：
> - `type=1` 表示员工（售后人员）
> - `type=2` 表示客户
>
> 系统会优先根据 `type` 字段进行身份分类，不再依赖 `job` 字段。

### 3.2 情感分析 (sentiment)

**summary**: 统计汇总
- `customer.good_reviews`: 客户好评数
- `customer.bad_reviews`: 客户差评数
- `employee.positive`: 员工积极消息数
- `employee.bad_attitude`: 员工恶劣态度消息数

**details**: 详细消息列表
- `customer_good`: 客户好评消息详情
- `customer_bad`: 客户差评消息详情
- `employee_positive`: 员工积极消息详情
- `employee_bad_attitude`: 员工恶劣态度消息详情

### 3.3 敏感词检测 (sensitive_words)

| 字段 | 类型 | 说明 |
| :--- | :--- | :--- |
| total_hits | int | 敏感词命中总次数 |
| words | array | 敏感词列表 |
| words[].word | string | 敏感词 |
| words[].count | int | 命中次数 |
| words[].hits | array | 命中详情 |

### 3.4 漏回分析 (unanswered_status)

| 字段 | 类型 | 说明 |
| :--- | :--- | :--- |
| is_missed | bool | 是否存在漏回 |
| decision_status | string | missed/answered/no_action_needed/pending/insufficient_data |
| risk_level | string | 风险等级：high/medium/low |
| missed_messages | array | 已核验的漏回客户原消息列表，包含 msgid、sender_name、sender_userid、sender_role、roomid、msgtime、content、time_source、waiting_minutes |
| suggested_action | string | 建议操作 |
| analysis_time | string | 分析执行时间；missed_messages[].msgtime 才是原消息发送时间 |
| explanation | string | 判定解释 |
| review_required | bool | 是否需要人工复核；模型失败时不会默认判漏回 |

### 3.5 高频词统计 (high_freq_words)

| 字段 | 类型 | 说明 |
| :--- | :--- | :--- |
| words | array | 高频词列表 |
| words[].word | string | 高频词 |
| words[].count | int | 出现次数 |
| words[].aliases | array | 同义词/别名 |

---

## 四、分析类型说明

| 类型值 | 说明 |
| :--- | :--- |
| sentiment | 情感分析（客户好评/差评、员工积极/恶劣） |
| sensitive | 敏感词检测 |
| summary | 群聊摘要生成 |
| highfreq | 高频词统计 |
| unanswered | 漏回消息分析 |

---

## 五、注意事项

1. **members字段**: Java侧需要在每条消息中传入群成员信息，格式为JSON字符串数组。如果某条消息已包含members，后续消息可以省略（系统会自动合并去重）。

2. **成员身份识别**: 系统使用 `type` 字段判断员工/客户身份（`type=1`为员工，`type=2`为客户），不再依赖 `job` 字段。请确保Java接口返回的 `type` 字段值正确。

3. **msgtype字段**: 消息类型，建议传 `"text"`，系统会优先分析文本类型消息。如果不传或传空，系统会自动使用所有有内容的消息进行分析。

4. **并发控制**: `max_concurrent`参数建议设置为5-10，不建议设置过大。

5. **消息数量**: 无最低限制，即使只有1条消息也会进行分析。

6. **字符编码**: 请确保使用UTF-8编码。

7. **超时设置**: 建议设置请求超时为300秒以上。

## 七、调用示例

### Java调用示例 (使用HttpClient)

```java
import java.net.URI;
import java.net.http.HttpClient;
import java.net.http.HttpRequest;
import java.net.http.HttpResponse;

public class ApiExample {
    public static void main(String[] args) throws Exception {
        String requestBody = """
        {
            "rooms": [{
                "room_id": "test_room",
                "room_name": "测试群",
                "messages": [{
                    "msgid": "msg_001",
                    "roomid": "test_room",
                    "from": "user_001",
                    "truename": "张三",
                    "content": "测试消息",
                    "msgtime": "2024-01-01 12:00:00",
                    "msgtype": "text",
                    "members": "[{\\"userid\\":\\"user_001\\",\\"name\\":\\"张三\\",\\"type\\":1}]"
                }]
            }],
            "analysis_type": ["sentiment", "sensitive", "summary", "highfreq", "unanswered"],
            "max_concurrent": 5
        }
        """;

        HttpClient client = HttpClient.newHttpClient();
        HttpRequest request = HttpRequest.newBuilder()
            .uri(URI.create("http://localhost:8000/api/v1/chat/batch-analyze"))
            .header("Content-Type", "application/json")
            .POST(HttpRequest.BodyPublishers.ofString(requestBody))
            .build();

        HttpResponse<String> response = client.send(request,
            HttpResponse.BodyHandlers.ofString());

        System.out.println(response.body());
    }
}
```
